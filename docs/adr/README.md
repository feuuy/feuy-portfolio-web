# Architectural Decision Records

Load-bearing architectural decisions. Created by the Tech Lead when a decision is
non-obvious OR a candidate was rejected for a load-bearing reason. Future explorers
read ADRs to avoid re-suggesting rejected approaches.

## Numbering

`NNNN-kebab-case-title.md`, zero-padded, monotonic.

## Template

```markdown
# NNNN. Title

**Status:** Proposed | Accepted | Deprecated | Superseded

## Context

## Decision

## Consequences
```

## Decision flow

Per `AGENTS.md`:

- Domain term → `CONTEXT.md`
- Architectural decision → `docs/adr/` (this folder)
- Work item → GitHub issue

---

This folder currently has no ADRs.
